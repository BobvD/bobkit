# Writing Plans

Use this skill after a spec exists and before implementation begins.

## Plan Requirements

- State the intended behavior change in one short summary.
- Group implementation work by subsystem or behavior.
- Include acceptance criteria and exact verification commands when known.
- Call out provider-specific or project-pack constraints.
- Keep tasks small enough to review independently.

## Plan Shape

Use `templates/plan.md` and `templates/tasks.md`.
