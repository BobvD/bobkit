# Spec-First Development

Use this skill when the goal needs clarification before implementation.

## Process

1. Restate the goal and the user-visible outcome.
2. Identify current behavior and constraints from the repository.
3. Draft a compact spec with success criteria, non-goals, risks, and verification needs.
4. Convert the approved spec into a task plan with clear file ownership and test expectations.
5. Do not implement until the spec and plan are specific enough for another agent or engineer to follow.

## Output

Use `templates/spec.md` for the spec and `templates/plan.md` for the implementation plan.
