# Feature Development Workflow

Use for new capabilities and meaningful behavior changes.

1. Activate `using-bobkit`.
2. Use `spec-first-development` to create a spec.
3. Use `writing-plans` to create implementation tasks.
4. Use `executing-tasks` to implement the approved tasks.
5. Use `test-driven-development` for testable behavior.
6. Use `code-review` for pre-merge review.
7. Use `verification-gate` before completion.
